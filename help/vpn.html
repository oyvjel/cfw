VPN tunneller
Cumulus firewall implementerer VPN som IPSEC kompatible tunneller mellom gatewayer. Maskinene krypterer og dekrypterer data til og fra subnett bak brannmurene, heretter kalt krypteringsdomene.

Dersom subnett bak brannmuren ikke spesifiseres er maskinens eksterne adresse selv hele krypteringsdomenet.

Som standard bør man velge RSA autentisering av tunellenes endepunkter.
Unntak:

PGPNet mobil PC

Autentisering: secret ( Preshared secret )
Lag secret som 0.0.0.0 freeswanGW-IP "mysecretkeywords" Definer ID = IP (0.0.0.0) i PGPNet

Linux FreeSWan mobil PC

Autentisering: rsasig ( Preshared secret virker ikke med variabel ( dhcp adresse) )
Definer ID = @something.pc på begge endepunkter.
Generer rsa nøkkelpar og utveksle offentlige nøkler.

RSA autentisering

Se http://www.strongsec.com/freeswan/ for hvordan du kan bruke x509 sertifikater.

Autentisering av tunnellenes endepunkter og generering av krypteringsnøkkel er basert på at hver maskin har et privat/offentlig nøkkelpar. Kun den offentlige nøklen er tilgjengelig fra konfigurasjonsbildene. Nøkkelparet identifiseres ved maskinens ID.

Generering av nøkler

Trykk på knappen Generer for å genererer ny nøkkel. Nøkkelparet plasseres i ..../vpn.secrets, og du kan returnere den offentlige nøklen til tunnel konfigurasjonen. Denne lokale offentlige nøklen må kopieres til alle "remote" gateways etter hver gang nytt nøkkelpar er generert. Tilsvarende må du få deres offentlige ID og nøkkel og legge den inn som Remote PUB Key.

Brannmurregler

For hver tunnel vil det bli generert nødvendige regler for å slippe selve tunneltrafikken gjennom brannmuren.

Dette inkluderer IKKE data som skal innkapsles i tunnellen. Det må defineres regler for å tillate aksess på vanlig måte.

NB. Remote nett blir nå tilknyttet virtuelt vpn interface ipsec?, og ikke reelt interface.
